Fields in generated metadata file and what they mean: The EFG import application generates a metadata file for each datasource. Authors have to edit this file to aid the application in making some decisions. In this write up the word 'Character(s)' and 'Field(s)' are sometimes used interchangeably. Some of these decisions are:
a. Which characters should appear on the search page.
b. The order of the characters on the search page
c. What to use as caption on search results page.
d. What characters should appear on the species pages
e. Which fields are images, similar taxa etc.
f. Which character states should be parsed into individual states

weight: This field is used by the application to order characters on the search page.(see http://efg.cs.umb.edu/efg/html/Redirect.jsp?dataSource=Ipo&renderer=pullDownService). The bigger the weight the higher it appears on the search page The weight is also used by the application to decide which caption should be used for display on the search results pages. The character with the biggest weight will be used as a caption for display on the search results page.

(see
http://alpaca.cs.umb.edu/efg/search?typeName=IpoData&maxDisplay=20). On the above page 'Scientific name' was used as the caption on the images because that was the character that the author choose to have the biggest weight. The default is zero.

name: (Does not require author edit).Is a list of the Characters in the current data source.

legal_name:(Does not require author edit) Is a list of names generated by the software.

searchable:(Requires author edit) Is a boolean field that indicates whether a character and its states will appear on the search page. 'true' means that author wants field to appear, 'false' means otherwise. The default is 'false'.

speciesPageData:(Requires author edit) Selecting 'true' means that author wants the character and it's states to be part of the generated species pages. (see: http://alpaca.cs.umb.edu/efg/search?Scientific name=Ipomoea batatas) for a sample of a species page. Defaults to 'false'.

holdsMultipleValues: (Requires author edit) Indicates whether a Character holds alternative states or not. 'true' means fields contain one or more alternatives, 'false' otherwise. Defaults to 'false'.

mediaResourceField: (Requires author edit) Check 'true' if the field is a Media resource field. False otherwise.

parserable: (Requires author edit)If the field contains alternative states that should be parsed. Example: Assume a taxonA with Character 'Wing Color' that has states 'blue,green,yellow,white' If author wants the field to be parsed into the individual colors then, this field should be checked as 'true'. Checking this field 'true' allows database searches to be done on the individual colors. So one can do a database search for all things with wing color 'blue' or 'white' for instance.

numeric (Requires author edit). If the 'field' holds numeric data. mark this field as 'true'. If you want numeric range data to be treated as numeric then mark it as true. For example: the states for a character are 2-10 mm and you want users to be able to get a hit when they enter '5'. Then the field should be marked as 'true'.
