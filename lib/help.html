Currently there is no way for an author to indicate the semantics of members of lists in EFG. For instance, a list of Similar Taxon could have a mixture of species, families and genera names. In making a query the application needs to know what each member of the list is. The EFG, does not allow the author to give that information to the application

BobMorris suggests that we adopt the following method:
name of field| value of field(comma separated);name of field| value of field; For example: Family|NAUCORIDAE,CALOPTERYGIDAE; Genus|Abedus, Argis,...;Species|Abedus xxx,Argis yyy....

Will appear in the same data(excel) cell. The fields 'Family', 'Genus', and 'Species' should already exists as fields

I think the above sentence assumes that descriptions (for the links in a list of related taxa) are in the same datasource as the current one. But this requirement is not imposed on the key software, and I doubt it needs to be imposed here. I guess it just comes to using the key mechanisms that do this. The interesting part is probably how do deal with failures to find the desired resources. Ideally, there could be a list of possible resources to query, similarly to Rod Paige's species server. (Put URL here) -- BobMorris - 08 Nov 2005

JenniferForman thinks this will be confusing to authors.
She suggests either authors split such fields into 3 fields: 'Similar Genera' , 'Similar Species', 'Similar Families' and enter the appropriate data using Bob's convention or

We do this at configuration time, and provide an interface for an author to associate
each member of the lists with the appropriate field name. A use case for such an interface could be as follows:
a. Author selects a field that contains a list of interest.
b. System offers author all available fields in the current data source.
c. Author selects one or more fields that should be used to query the database with a value
from the lists in a. above.
d. System stores information and uses it at run time.

So lets assume that a cell for a field X has the following data: NAUCORIDAE,CALOPTERYGIDAE,Abedus, Argis,Abedus xxx, Argis yyy Lets also assume that the configuartion has been done per the use case above.

For each member of the list, a query will have to be made with 'Family', 'Genus' and 'Species' as follows: Family=NAUCORIDAE or Genus=NAUCORIDAE or Species = NAUCORIDAE or Family =CALOPTERYGIDAE .....

Comments are welcome.

The stuff that comes before the '|' could be a URL or a partial URL to a service that understands what to do with a string that is sent to it. For instance: 'http://alpaca.cs.umb.edu/efg/search?Family=|'..EFG appends the current Family name to it and sends the query to a service located at the URL that understands how to handle it.

-- JacobAsiedu - 08 Nov 2005
